# 產品需求文件 (PRD) - 食譜收藏夾系統

## 1. 專案概述
### 背景與動機
目前市面上的食譜多散落在各大網路平台（部落格、YouTube、IG等），個人的實作心得也往往散落在紙本筆記或是不好管理的備忘錄中。這導致在真正要進廚房料理時，經常手忙腳亂去翻找資料。本專案旨在打造一個專門為個人設計的數位食譜筆記本，將網路上散亂的靈感與自己的實作心得，整合進一個視覺簡約、易於操作的結構化空間。

### 目標用戶
個人使用者（如開發者本人），喜好料理且有收集與整理實作心得需求的使用者。

### 核心價值主張
提供一個簡約、結構化且具行動版閱讀優化的個人食譜管理空間，讓烹飪變得優雅且有條理，並能隨時間累積個人的私房秘方與調整心得。

## 2. 功能需求

### 主要功能清單與使用者故事

1. **結構化食譜輸入**
   - **使用者故事**：作為使用者，我希望能將食譜拆分成「食材清單」、「烹飪步驟」與「私房筆記」等欄位輸入，以便在閱讀時不會感到雜亂無章。
2. **多維度分類標籤**
   - **使用者故事**：作為使用者，我希望能為食譜加上自定義標籤（如：減脂、15分鐘出菜、氣炸鍋），以便在未來能快速透過標籤交叉篩選想做的菜。
3. **行動版閱讀優化**
   - **使用者故事**：作為使用者，我希望在手機上閱讀食譜時能有大按鈕與清晰的字體，以便我在廚房一邊操作一邊觀看時，能輕鬆看清楚步驟與份量。
4. **關鍵字快速檢索**
   - **使用者故事**：作為使用者，我希望能輸入現有的食材關鍵字（如：雞蛋、番茄），以便系統能立即找出現有可製作的相關食譜。
5. **烹飪心得紀錄**
   - **使用者故事**：作為使用者，我希望能針對每次實作留下評分與微調記錄，以便記下口味（如：鹹淡調整）或火候心得，讓下一次料理更完美。

## 3. 非功能需求
### 技術限制
- **後端開發語言與框架**：Python + Flask
- **前端模板引擎**：Jinja2
- **資料庫儲存**：SQLite
- **前端樣式**：原生 CSS 或輕量級 CSS 框架，著重行動版優先體驗。

### 效能與安全考量
- **效能**：考量為個人單機或輕量化系統，確保頁面載入速度快，即便食譜圖文增加，檢索與載入應在 1 秒內完成。
- **安全**：預設基本的防護機制，若於外網或雲端運行則需加入帳號密碼保護。輸入區需防範常見的 XSS (Cross-Site Scripting) 及 SQL Injection 攻擊，確保筆記內容安全。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必備範圍，第一階段上線)
- [ ] 食譜的 CRUD 功能（新增、讀取、編輯與刪除），並區分「食材清單」、「烹飪步驟」、「私房筆記」欄位。
- [ ] 行動版閱讀優化的清單與食譜瀏覽頁面。
- [ ] 基礎的關鍵字（標題/食材）搜尋功能。

### Should Have (應備功能，可做為下一階段更新)
- [ ] 可自定義的多維度標籤系統（例如：減脂、15分鐘）及標籤過濾。
- [ ] 獨立的「烹飪心得紀錄」欄位或版本系統，可為每次下廚新增紀錄與調整備註。

### Nice to Have (加分項目，未來規畫)
- [ ] 貼上網址自動解析網頁食譜或匯入資料的功能。
- [ ] 圖片上傳與預覽支援。
